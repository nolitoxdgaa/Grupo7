# Requisitos de Usuario

Los **requisitos de usuario** establecen las **necesidades, expectativas y capacidades** que debe satisfacer el sistema **3D PC Builder** desde la perspectiva del usuario final.  
Se enfocan en **cómo el usuario interactúa con la aplicación**, qué información debe recibir y qué acciones debe poder realizar dentro del entorno de simulación 3D.

## Lista ampliada de requisitos de usuario

1. **Exploración de componentes.**  
   El usuario debe poder visualizar todos los componentes disponibles (CPU, GPU, RAM, almacenamiento, fuente de poder, gabinete, etc.) de forma organizada y filtrable por categoría o marca.

2. **Consulta de información detallada.**  
   El sistema debe permitir al usuario acceder a especificaciones técnicas, precios, rendimiento y descripciones de cada componente de manera clara y legible.

3. **Interacción con modelos tridimensionales.**  
   El usuario debe poder manipular los modelos 3D con controles intuitivos para rotar, acercar, alejar y examinar cada pieza desde distintos ángulos.

4. **Construcción y personalización de ensamblajes.**  
   El usuario debe tener la posibilidad de arrastrar y soltar los componentes dentro de un gabinete 3D, creando configuraciones personalizadas.

5. **Verificación de compatibilidad.**  
   El sistema debe analizar las piezas seleccionadas y notificar al usuario si existen incompatibilidades (por ejemplo, procesador incompatible con la placa base).

6. **Recepción de alertas y mensajes informativos.**  
   Cuando se detecten errores o conflictos, el usuario debe recibir alertas visuales o mensajes explicativos con posibles soluciones o recomendaciones.

7. **Visualización del costo total del ensamblaje.**  
   El usuario debe poder ver el precio individual de cada componente y el costo total actualizado conforme realiza modificaciones en el ensamblaje.

8. **Gestión de configuraciones.**  
   El usuario debe poder guardar sus ensamblajes terminados, cargarlos nuevamente en otra sesión o exportarlos a formatos como PDF o CSV.

9. **Simulación de ensamblajes personalizados.**  
   El sistema debe permitir al usuario probar distintas combinaciones de componentes y visualizar el resultado final en el entorno 3D.

10. **Asistencia y retroalimentación.**  
   El sistema debe ofrecer mensajes de ayuda, consejos o advertencias de seguridad que orienten al usuario durante el proceso de ensamblaje.

11. **Personalización de la experiencia.**  
    El usuario debe poder ajustar opciones del entorno 3D, como zoom, color de fondo o modo de vista, para adaptar la experiencia a sus preferencias.

12. **Rendimiento fluido durante la simulación.**  
    El usuario debe experimentar una navegación y renderización estable, con al menos **30 FPS** durante la visualización del modelo 3D.

13. **Ejecución sin conexión a Internet.**  
    El usuario debe poder usar la aplicación sin requerir conexión en línea, accediendo a todos los recursos principales desde su dispositivo local.

14. **Compatibilidad con equipos de gama media.**  
    El usuario debe poder ejecutar el programa en computadoras con hardware estándar (por ejemplo, GPU RTX 3060 o equivalente) sin pérdidas notables de rendimiento.

15. **Interfaz moderna y fácil de usar.**  
    La interfaz debe ser limpia, ordenada y comprensible incluso para usuarios con poca experiencia técnica, con iconos y botones fácilmente identificables.

16. **Retroalimentación visual inmediata.**  
    El sistema debe responder de forma rápida a las acciones del usuario (por ejemplo, resaltar un componente seleccionado o mostrar animaciones suaves al ensamblar).

19. **Seguridad en la manipulación virtual.**  
    El sistema debe advertir al usuario sobre malas prácticas o combinaciones potencialmente riesgosas en un PC real (por ejemplo, falta de fuente de poder suficiente).

20. **Satisfacción y experiencia positiva.**  
    El usuario debe percibir una experiencia educativa, fluida y atractiva, con sensación de control total sobre el proceso de diseño y simulación.
