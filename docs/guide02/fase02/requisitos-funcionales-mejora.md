# Requisitos Funcionales de Mejora

Los **requisitos funcionales de mejora** representan características que **amplían o complementan las funcionalidades mandatorias** del sistema **3D PC Builder**.  
No son indispensables para el **MVP**, pero su incorporación en futuras versiones aumentará la usabilidad, el realismo y el valor del producto.

---

## Lista de requisitos de mejora

1. **Incorporación de ejercicios prácticos con presupuestos simulados.**  
   Permitir que el usuario practique la construcción de PCs con **presupuestos límite**, de manera que pueda:
   - Ajustar componentes según restricciones económicas.  
   - Evaluar distintas combinaciones rendimiento/precio.  

2. **Integración de precios actualizados desde distintos proveedores.**  
   En versiones posteriores (no en el MVP), conectar el sistema con **fuentes externas** (APIs o bases de datos de tiendas especializadas) para:
   - Mostrar precios aproximados o simulados de los componentes.  
   - Realizar comparaciones de costo entre configuraciones.  

3. **Recomendaciones automáticas de componentes compatibles.**  
   Sugerir **alternativas de componentes** en función del hardware ya seleccionado, por ejemplo:
   - Placas base compatibles con un procesador elegido.  
   - Fuentes de poder adecuadas al consumo total del sistema.  
   - Opciones de RAM compatibles con la placa base y el uso previsto.

4. **Gestión y simulación de configuraciones personalizadas.**  
   Permitir que el usuario:
   - Guarde configuraciones de ensamblaje creadas previamente.  
   - Cargue configuraciones guardadas para modificarlas o revisarlas.  
   - Duplique configuraciones para compararlas entre sí.

5. **Modo de exploración de componentes sin ensamblar.**  
   Incorporar un modo en el cual el usuario pueda **explorar libremente la biblioteca de componentes**, sin necesidad de armar una PC completa, revisando:
   - Especificaciones.  
   - Posibles combinaciones recomendadas.  

6. **Interfaz más moderna y enfocada en la experiencia de usuario (UX).**  
   En futuras iteraciones, refinar la interfaz gráfica para:
   - Mejorar la organización visual de la información.  
   - Simplificar los flujos de navegación.  
   - Incluir elementos gráficos y layouts más consistentes con buenas prácticas de UI/UX.
