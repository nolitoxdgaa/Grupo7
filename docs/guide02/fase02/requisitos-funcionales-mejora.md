# Requisitos Funcionales de Mejora

Los **requisitos funcionales de mejora** representan aquellas características que **complementan o amplían las funcionalidades mandatorias** del sistema **3D PC Builder**.  
No son indispensables para la primera versión del software, pero su incorporación en futuras iteraciones aumentará la usabilidad, realismo y valor general del producto.

---

## Lista de requisitos de mejora

1. **Incorporación de ejercicios prácticos con presupuestos simulados.**  
   Permitir al usuario practicar la construcción de PCs con presupuestos límite, evaluando su conocimiento y capacidad de optimización de componentes.

2. **Integración de precios actualizados desde distintos proveedores.**  
   Conectar el sistema con bases de datos o APIs de tiendas especializadas para mostrar precios reales y comparativos.

3. **Cálculo automático del costo total del ensamblaje.**  
   Generar el costo total en tiempo real conforme se agregan o eliminan componentes durante el ensamblaje virtual.

4. **Recomendaciones automáticas de componentes compatibles.**  
   Sugerir alternativas o piezas complementarias según el hardware seleccionado, optimizando el rendimiento y la compatibilidad.

5. **Simulación de diseños personalizados creados por el usuario.**  
   Permitir importar o guardar diseños 3D propios para visualizar configuraciones únicas dentro del entorno del sistema.

6. **Interfaz moderna y enfocada en la experiencia de usuario (UX).**  
   Mejorar la estética, fluidez e interacción general del sistema mediante una interfaz más intuitiva y adaptada a las buenas prácticas de diseño UI/UX.
